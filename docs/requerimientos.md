### 1. Requerimientos Funcionales (RF)
Definen las acciones que el sistema debe realizar:
*   **RF-01: Registro de Profesionales.** El sistema debe permitir a los trabajadores de oficio registrar su perfil (nombre, oficio, contacto).
*   **RF-02: Búsqueda Pública.** Los ciudadanos deben poder buscar trabajadores por oficio o palabra clave.
*   **RF-03: Visualización de Perfiles.** El sistema debe mostrar los datos de contacto de los profesionales encontrados.

### 2. Requerimientos No Funcionales (RNF)
Definen criterios de calidad y restricciones:
*   **RNF-01: Diseño Responsive.** La interfaz debe ser intuitiva y adaptarse a dispositivos móviles (*Mobile-first*).
*   **RNF-02: Persistencia de Datos.** Toda la información registrada debe almacenarse en una base de datos real, no solo en memoria volátil.
*   **RNF-03: Disponibilidad.** La aplicación debe ser accesible públicamente a través de una URL en la nube.
